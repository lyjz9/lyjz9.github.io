Goal

Job details live across different boards and company pages, which makes manual tracking repetitive.

02

Build

Created a local web app that accepts posting links, extracts key fields, normalizes labels, and writes rows into Excel.

03

Result

Added review flags so uncertain results are easy to check instead of silently trusting every scrape.

build notes

Choices and small problems

  • Kept the original job link attached so every extracted row can be checked later.
  • Added review flags because scraping can be blocked, incomplete, or weird across different job boards.
  • Wrote rows into the tracker without replacing the workbook, so existing notes stay intact.
  • Built it as a local workflow first because job sites change often and need hands-on testing.
