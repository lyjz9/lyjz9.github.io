Synthetic insurance data / frequency analysis

01

Goal

Driver age may relate to claim frequency, but claim counts need a model that respects count-data behavior.

02

Build

Created a Python workflow that generates synthetic data, engineers features, compares age groups, and fits Poisson and Negative Binomial models.

03

Result

Saved refreshed figures, summary tables, model output, and a public-ready README for a clean portfolio repo.

build notes

Choices and small problems

  • Used synthetic claim data so the modeling flow can be shared without real insurance records.
  • Compared Poisson and Negative Binomial models because claim counts can be overdispersed.
  • Kept the age-group finding readable while noting that age alone is not enough for real pricing.
  • Saved model tables and figures so the project can be checked without rerunning everything first.
